## 问题分析

当前签到流程的问题：
1. 页面加载后自动检查摄像头权限
2. 如果用户之前已授权，会立即启动摄像头
3. 用户还没准备好就被摄像头打扰，体验不好

## 优化方案

### 1. QrScanner 组件修改
- 添加 `manual` 模式（默认）
- 默认显示"开始扫码"按钮，不自动启动摄像头
- 用户点击按钮后才请求权限并启动摄像头
- 保留自动模式作为可选配置

### 2. 新增状态：准备扫码
- 显示二维码图标和"开始扫码"按钮
- 点击后才进入权限请求/摄像头启动流程

### 3. 流程优化
```
页面加载 → 显示"开始扫码"按钮 → 用户点击 → 请求权限 → 启动摄像头
```

### 具体修改

**QrScanner.vue:**
1. 添加 `manual` prop 控制是否手动启动
2. 新增 `ready` 状态，默认显示扫码入口
3. 用户点击"开始扫码"后才检查权限/启动摄像头
4. 扫码成功后可以重新开始扫码

**UI 改进:**
- 准备状态：显示二维码图标 + "点击开始扫码"按钮
- 权限请求：保持现有设计
- 扫码中：保持现有设计

这样用户进入签到页面后，需要主动点击按钮才会启动摄像头，体验更加友好。
