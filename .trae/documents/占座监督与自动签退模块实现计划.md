## 占座监督与自动签退模块实现计划

### 项目现状分析
- **前端**: Vue 3 + TypeScript + Ant Design Vue + Pinia + WebSocket
- **后端**: Spring Boot 3 + MyBatis-Plus + MySQL + Redis + WebSocket
- **已有功能**: 预约、签到(GPS+二维码)、暂离、违约检测、信用分系统

### 设计方案

#### 一、占座监督模块
1. **数据库表**: 新增 `sys_seat_occupancy` 表记录占座检测数据
2. **核心功能**:
   - 定时检测用户签到后的离开时长
   - 超过阈值(默认60分钟)触发占座预警
   - 预警后5分钟未返回则自动签退
3. **配置项**:
   - `occupancy_check_interval`: 检测间隔(分钟)
   - `occupancy_threshold`: 占座阈值(分钟)
   - `occupancy_warning_time`: 预警时间(分钟)

#### 二、自动签退模块
1. **触发条件**:
   - 占座超时(扣15信用分)
   - 预约时段结束(正常完成+2分)
   - 图书馆闭馆(22:00自动签退)
2. **定时任务增强**:
   - 每5分钟执行占座检测
   - 每日21:30发送闭馆提醒
   - 每日22:00执行闭馆自动签退

#### 三、实现阶段

**Phase 1: 数据库设计与配置**
- 创建 `sys_seat_occupancy` 表
- 添加系统配置项到 `sys_config` 表

**Phase 2: 后端核心服务开发**
- 创建 `SeatOccupancy` Entity 和 Mapper
- 创建 `OccupancyMonitorService` 服务类
- 创建 `OccupancyController` 控制器
- 实现占座检测逻辑

**Phase 3: 定时任务增强**
- 增强 `ReservationJob` 添加占座检测方法
- 添加闭馆提醒和自动签退定时任务
- 实现自动签退逻辑(更新预约状态、释放座位、扣除信用分)

**Phase 4: 前端监控页面开发**

**Phase 5: 测试与优化**
